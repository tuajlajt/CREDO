## Model Overview

MedGemma 1.5 is Google's vision-language model collection for healthcare, released
January 2026 as part of HAI-DEF. Built on Gemma 3, it includes:

- `google/medgemma-4b-it` — 4B multimodal (image + text input, text output)
  Use for: radiology image Q&A, image captioning, visual question answering
- `google/medgemma-27b-it` — 27B text-only
  Use for: long clinical document understanding, complex clinical reasoning, EHR analysis
- `google/medgemma-27b-it-medqa` — fine-tuned on medical Q&A benchmarks

Primary use cases: answering image-based medical questions, drafting radiology reports,
extracting clinical data from unstructured text, pre-visit information gathering.

**Important**: MedGemma outputs are not validated for direct clinical use without
human expert review. Always include disclaimers and route through domain-expert.

## Standard Inference

```python
# src/models/medgemma/inference.py
from transformers import AutoProcessor, AutoModelForImageTextToText
import torch
from PIL import Image

def load_medgemma(model_id: str = "google/medgemma-4b-it",
                  device: str = "auto") -> tuple:
    """Load MedGemma processor and model."""
    processor = AutoProcessor.from_pretrained(model_id)
    model = AutoModelForImageTextToText.from_pretrained(
        model_id,
        torch_dtype=torch.bfloat16,
        device_map=device,
    )
    model.eval()
    return processor, model


def analyze_medical_image(
    processor, model,
    image: Image.Image,
    question: str,
    max_new_tokens: int = 512,
) -> str:
    """
    Run MedGemma on a medical image + question.
    Input image must be de-identified before calling this function.
    """
    messages = [
        {
            "role": "user",
            "content": [
                {"type": "image", "image": image},
                {"type": "text",  "text": question},
            ]
        }
    ]
    inputs = processor.apply_chat_template(
        messages, add_generation_prompt=True,
        tokenize=True, return_tensors="pt"
    ).to(model.device)

    with torch.inference_mode():
        output = model.generate(**inputs, max_new_tokens=max_new_tokens)

    # Decode only new tokens
    response = processor.decode(
        output[0][inputs["input_ids"].shape[-1]:],
        skip_special_tokens=True
    )
    return response
```

## Radiology Report Generation

```python
RADIOLOGY_PROMPT = """You are a radiologist assistant. Analyze this {modality} image
and provide a structured report with the following sections:
- Technique
- Findings
- Impression

Note any significant abnormalities. Be specific about location, size, and character
of findings. Do not include patient identifiers."""

def generate_radiology_report(processor, model, image: Image.Image,
                               modality: str = "chest X-ray") -> str:
    prompt = RADIOLOGY_PROMPT.format(modality=modality)
    return analyze_medical_image(processor, model, image, prompt)
```

## Limitations and Safety

- Outputs must be reviewed by a qualified clinician before clinical use
- Model can hallucinate — unusual findings need human verification
- Not validated for rare conditions or paediatric imaging
- Always append: "This report was generated by AI and requires radiologist review."
- Log all generated reports with timestamp, model version, and prompt used

## Red Flags

- Using MedGemma output directly in clinical workflow without human review
- Passing PHI in the image or prompt
- Temperature > 0.3 for clinical report generation (increases hallucination)
- Not logging model version with each generated report
